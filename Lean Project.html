Lead: Surjya Narayan Sabat
Date: October 2024

Business Case

During the shared drive migration process, small finance and production automations were severely impacted due to the reliance on an outdated database system (MS Access). Over 1,200 associates faced critical challenges, including performance bottlenecks, limited scalability, security vulnerabilities, and financial losses. These issues stem from dependence on obsolete infrastructure, directly affecting operational efficiency and hampering the Workforce Management (WFM) application, back-office inquiries, claims processing, complaint handling, and strategic decision-making. Additionally, the shared drive environment posed significant security risks, such as exposure to accidental file deletions, unauthorized access, and data corruption. These vulnerabilities disrupted management operations and compromised the ability to deliver real-time reporting for leadership decision-making.

Problem Statement

  • • Performance Issues: Processing a single case on finance automation takes 6–7 minutes, leading to inefficiencies in WFM and other tools, thereby lowering productivity.
  • • Scalability Constraints: The system can support only 10–12 reliable concurrent connections, limiting collaboration and operational efficiency.
  • • Dependency on Legacy Systems: Reliance on Windows 7 restricts compatibility with modern tools and systems, delaying innovation and growth.
  • • Efficiency: Half a day is lost validating/consolidating data from 100+ MS Access databases.

Project Conclusion

By successfully applying Lean Six Sigma principles through the PDCA cycle, this project transitioned critical operations from a fragile, legacy MS Access environment to a robust, scalable Azure SQL cloud infrastructure. The initiative eliminated performance bottlenecks impacting 1,200 associates and achieved over 80% reduction in manual effort. Power BI integration now provides real-time data-driven insights, ensuring a sustainable foundation for future growth.
